# ai-dock — Specification

A macOS menu-bar app that surfaces Codex rate-limit status and OpenRouter credit balance as a glanceable status item. This document is the build spec; product reasoning and decision history live in `DESIGN.md` and `docs/adr/`.

---

## 1. Product behavior

### 1.1 App shape

A menu-bar-only application. No dock icon, no main window, no app in the Cmd-Tab switcher. Set via macOS `LSUIElement = true` (Tauri: `Activation Policy accessory` at runtime, or the `Info.plist` key).

The app exposes:
- A **tray icon** in the macOS menu bar (top-right).
- A **popover** — a hidden borderless window, shown when the tray icon is clicked, repositioned under the icon.
- A **settings affordance** inside the popover for the OpenRouter key.

### 1.2 Tray icon

Static. No ambient signal, no fill, no ring, no badge. The icon never changes state based on data. All detail lives in the popover.

A real tray icon asset is required (the Tauri default is a placeholder). Exact design TBD; a simple monochrome glyph is sufficient for v1.

### 1.3 Popover lifecycle

- **Open:** tray icon click.
- **Close:** window blur (click elsewhere, Cmd-Tab, click another app), Escape key, or tray icon click while open (toggle).
- **Flicker guard:** a just-toggled flag prevents close-then-immediate-reopen when the tray icon is clicked while open (the click fires during the blur event).
- **No auto-timeout.** The popover stays open until the user dismisses it.

### 1.4 Popover content

Three rows, in order:

| Row | Source | Display |
|-----|--------|---------|
| Codex Primary window | `logs_2.sqlite` | Filled bar of `used_percent` + reset timestamp |
| Codex Secondary window | `logs_2.sqlite` | Filled bar of `used_percent` + reset timestamp |
| OpenRouter credits | `/api/v1/credits` | Differentiated bar (distinct treatment from Codex) + `$used / $total` text |

Plus:
- A **manual refresh button** that forces an immediate poll of both sources.
- A **settings affordance** (gear icon or "Settings" row) that opens a small field to enter/edit the OpenRouter key.

### 1.5 No auto-launch

The app is not added to macOS login items. No `tauri-plugin-autostart` dependency. The user launches it manually.

## 2. Data sources

### 2.1 Codex — `logs_2.sqlite`

**Path:** `~/.codex/logs_2.sqlite` (resolved via `dirs::home_dir()`).

**Schema** (verified on the target machine):

```sql
CREATE TABLE logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  ts INTEGER NOT NULL,              -- unix epoch seconds
  ts_nanos INTEGER NOT NULL,
  level TEXT NOT NULL,
  target TEXT NOT NULL,
  feedback_log_body TEXT,           -- contains the JSON event, prefixed by tracing output
  ...
);
```

**Signal:** Codex logs a structured `codex.rate_limits` websocket event on every turn. The event appears as a JSON object embedded in `feedback_log_body`, prefixed by tracing span text. Verified shape:

```json
{
  "type": "codex.rate_limits",
  "plan_type": "plus",
  "rate_limits": {
    "allowed": true,
    "limit_reached": false,
    "primary": {
      "used_percent": 1,
      "window_minutes": 300,
      "reset_after_seconds": 18000,
      "reset_at": 1782162131
    },
    "secondary": {
      "used_percent": 28,
      "window_minutes": 10080,
      "reset_after_seconds": 233556,
      "reset_at": 1782385000
    },
    "code_review_rate_limits": null,
    "additional_rate_limits": null,
    "credits": null,
    "promo": null
  }
}
```

**Extraction query** (read-only, no writes):

```sql
SELECT feedback_log_body
FROM logs
WHERE feedback_log_body LIKE '%codex.rate_limits%'
ORDER BY ts DESC, ts_nanos DESC, id DESC
LIMIT 1;
```

**Parse approach:** locate the substring `{"type":"codex.rate_limits"` in `feedback_log_body`, then parse balanced braces forward to the matching closing `}`. Do not rely on a trailing marker or greedy regex (the body has tracing prefixes/suffixes). Parse with `serde_json`.

**Rust struct shape (the parser must mirror the real nesting):** `primary` and `secondary` are *not* at the top level — they are nested under `rate_limits`. A flat struct expecting top-level `primary`/`secondary` fails with `missing field primary`. The deserializer must be:

```rust
struct CodexRateLimitEvent {
    plan_type: String,
    rate_limits: RateLimits, // wrapper
}
struct RateLimits {
    primary: CodexWindow,
    secondary: CodexWindow,
}
```

serde ignores the other top-level fields (`type`, `allowed`, `limit_reached`, `code_review_rate_limits`, `additional_rate_limits`, `credits`, `promo`) by default — no `#[serde(deny_unknown_fields)]`.

**Reset-time normalization (required):** the event's `reset_at` and `used_percent` are a snapshot from the last Codex turn. If `reset_at <= now`, that rolling window has already rolled — the stale `used_percent` no longer applies. Before emitting, the Rust side *must* zero out `used_percent` and `reset_after_seconds` for any window whose `reset_at <= now`, so the popover never shows a limit that's already reset. The frontend renders a zeroed window as a 0% bar + past-tense "reset" (not "resets now"). The next Codex turn emits a fresh event with real current usage.

**Fields consumed:**
- `plan_type` → badge in the popover (e.g. "Plus").
- `rate_limits.primary.used_percent` → Primary bar fill.
- `rate_limits.primary.reset_at` → Primary reset timestamp.
- `rate_limits.primary.window_minutes` → confirms 300 (5h); displayed as "5h window" if shown.
- `rate_limits.secondary.*` → Secondary bar (same fields).
- `rate_limits.allowed`, `rate_limits.limit_reached` → available for error/stale decisions (though Q9 chose a static icon, these remain in the payload for future use).

**No auth.** The Codex side does not read `~/.codex/auth.json`. See ADR-0001.

**Connection mode:** read-only. Open with `OpenFlags::SQLITE_OPEN_READ_ONLY | SQLITE_OPEN_NO_MUTEX`. Use the WAL mode the DB already has (allows concurrent reads while Codex writes). Do not hold the connection across polls — open, query, close, each cycle.

### 2.2 OpenRouter — `/api/v1/credits`

**Endpoint:** `GET https://openrouter.ai/api/v1/credits`

**Auth:** `Authorization: Bearer <management_key>`. The docs flag this as a management-key endpoint; a standard chat `sk-or-...` key may be rejected with 401/403. The error policy (§4.2) handles this.

**Response shape** (verified from OpenRouter docs):

```json
{
  "data": {
    "total_credits": 100.5,
    "total_usage": 25.75
  }
}
```

**Derived value:** remaining = `total_credits - total_usage` (the API does not return a "remaining" field).

**HTTP client:** `reqwest` (blocking or async depending on runtime; the poll loop is async via tokio, so `reqwest` async).

**Timeout:** 10 seconds. Failures fall through to the error policy (§4.2).

## 3. Credential storage

**Path:** `~/.config/ai-dock/config.json` (resolved via `dirs::config_dir()`).

**Format:** JSON (serde_json is already a Tauri dependency; no new crate for TOML).

**Shape:**

```json
{
  "openrouter_key": "sk-or-..."
}
```

**Behavior:**
- Created lazily on first write (when the user enters a key in settings). Not created at launch if absent.
- Read by Rust at poll time. The key never crosses to the frontend webview.
- Hand-editable; the app re-reads on each poll (no in-memory cache that goes stale after an external edit).
- Missing file or missing field → treated as a persistent error ("no OpenRouter key — add one in settings").

## 4. Error and stale states

**Invariant:** never render a number we don't have. Zero is a number. A failed fetch is not zero and must not be rendered as zero.

### 4.1 Codex failure modes

| Failure | Display |
|---------|---------|
| `logs_2.sqlite` missing or can't open | Error caption: "Codex: no logs found — run `codex` once." No bar. |
| DB locked (Codex mid-write) | Dim last-known bar, caption: "Codex: updating…" Retry next poll. |
| No `codex.rate_limits` rows (DB exists, no events) | Error caption: "Codex: no rate-limit data yet." No bar. |
| JSON parse fail / schema changed | Error caption: "Codex: log format changed — check for app update." No bar. |

### 4.2 OpenRouter failure modes

| Failure | Display |
|---------|---------|
| 401/403 (key missing, wrong, or chat key not management key) | Error caption: "OpenRouter: key rejected — is it a management key?" No bar. |
| 429 (rate-limited by OpenRouter) | Dim last-known bar, caption: "OpenRouter: rate-limited, retrying." Retry next poll. |
| Network error / timeout / 5xx | Dim last-known bar, caption: "OpenRouter: couldn't fetch." Retry next poll. |
| Malformed JSON | Dim last-known bar, caption: "OpenRouter: unexpected response." Retry next poll. |
| No key configured | Error caption: "OpenRouter: no key — add one in settings." No bar. |

### 4.3 First-launch state (no prior good values)

When there has never been a good value for a source, there is nothing to show stale. The row renders the error caption only, no bar. This is the honest first-run experience: empty popover with two "no data" rows and a settings prompt.

## 5. Architecture

### 5.1 Rust owns the poll loop

A background `tokio` task polls every **5 minutes** (300 seconds), same interval for both sources. Each cycle:

1. Read Codex `logs_2.sqlite` → parse latest `codex.rate_limits` event.
2. Read `~/.config/ai-dock/config.json` → if key present, `GET /api/v1/credits`.
3. Build a combined `Status` payload.
4. Emit a `status-update` Tauri event to the frontend.

The frontend never polls, never fetches, never reads files. It subscribes to `status-update` and renders.

### 5.2 Manual refresh

A Tauri command `refresh_now()` that the frontend invokes from the refresh button. It triggers an immediate poll cycle (same logic as the timer) and emits `status-update`. The 5-min timer is not reset by a manual refresh (the next auto-poll fires on its own schedule).

### 5.3 Tray + popover plumbing

- `TrayIconBuilder` creates the tray icon with the static asset.
- On tray click: if the popover window is hidden, reposition it under the tray icon and show; if visible, hide. (Toggle.)
- The popover window is created at app launch with `visible: false`, borderless, no decorations, `always_on_top`, `skip_taskbar` (Tauri equivalents).
- Repositioning under the tray icon: query the tray icon's position (Tauri API) and set the window position so the popover appears below it, account for the popover's width/height.
- Hide-on-blur: `window.on_focus_changed` → hide when focus lost, *unless* the just-toggled flag is set (set on tray click, cleared after a short delay or the next event loop tick).

### 5.4 macOS-specific config

- `LSUIElement = true` (or `Activation Policy accessory` at runtime) — no dock icon, no Cmd-Tab entry.
- The window is borderless (`decorations: false`) and transparent-background-capable if the popover design needs rounded corners / a shadow.

## 6. Payload contract (Rust → frontend)

The `status-update` event payload:

```ts
type StatusUpdate = {
  codex: CodexStatus;
  openrouter: OpenRouterStatus;
  polled_at: number; // unix epoch seconds
};

type CodexStatus =
  | { kind: "ok"; plan_type: string; primary: WindowStatus; secondary: WindowStatus; event_ts: number }
  | { kind: "error"; message: string }
  | { kind: "stale"; plan_type: string; primary: WindowStatus; secondary: WindowStatus; event_ts: number; message: string };

type WindowStatus = {
  used_percent: number;  // zeroed by Rust if reset_at <= now (window already rolled)
  window_minutes: number;
  reset_after_seconds: number;  // zeroed by Rust if reset_at <= now
  reset_at: number; // unix epoch seconds
};

type OpenRouterStatus =
  | { kind: "ok"; total_credits: number; total_usage: number; remaining: number }
  | { kind: "error"; message: string }
  | { kind: "stale"; total_credits: number; total_usage: number; remaining: number; message: string };
```

`kind` discriminates the three display states (§4): `ok` (normal), `stale` (dim + caption), `error` (caption only, no bar).

## 7. Frontend

### 7.1 Stack

React 19 + TypeScript + Vite, kept from the boilerplate. No new frontend dependencies for v1.

### 7.2 Components

```
src/
  App.tsx                      # subscribes to status-update, renders popover shell
  App.css                      # popover styles (replaces boilerplate)
  components/
    Bar.tsx                    # the progress bar; prop: variant ("filled" | "differentiated"), fill (0-100)
    Row.tsx                    # one row: label + bar + secondary text + caption (stale/error)
    Settings.tsx               # the OpenRouter key entry field; calls invoke("set_openrouter_key", ...)
```

### 7.3 Bar variants

- **Codex rows:** `variant="filled"`. A solid filled bar, color escalates near the cap (thresholds TBD at implementation; default: amber at 80%, red at 95%).
- **OpenRouter row:** `variant="differentiated"`. Same shape (a bar), distinct treatment — outline/ghost or neutral color, TBD at implementation per ADR-0003. Fill ratio = `total_usage / total_credits`.

### 7.4 Event subscription

`App.tsx` listens to the Tauri `status-update` event on mount, stores the latest payload in state, and passes slices to the rows. On unmount, unsubscribe.

### 7.5 Refresh button

Calls `invoke("refresh_now")`. The actual UI update comes through the same `status-update` event (the manual refresh triggers a poll that emits the event); the button does not directly update state.

## 8. Tauri configuration changes

### 8.1 `tauri.conf.json`

- `"productName"`: keep `ai-docked` or rename to `ai-dock` (consistency with repo name).
- `"app.windows[0]"`: set `"visible": false`, `"decorations": false`, `"alwaysOnTop": true`, `"skipTaskbar": true`, `"resizable": false`. Reduce `width`/`height` to popover dimensions (e.g. 320 × 240).
- Add tray icon configuration if Tauri 2 supports it via config (otherwise build in Rust via `TrayIconBuilder`).

### 8.2 `capabilities/default.json`

- Add permission for the `tray` plugin / tray icon if required by the Tauri 2 tray API.
- Add permission to emit events to the window (`core:event:allow-emit` or equivalent).
- Remove `opener:default` (we're cutting `tauri-plugin-opener`).

### 8.3 `Cargo.toml` — dependency changes

Add:
- `tokio` (with `rt-multi-thread`, `time` features — for the poll interval)
- `reqwest` (default features or `json` feature — for OpenRouter fetch)
- `rusqlite` (with `bundled` feature — for reading `logs_2.sqlite`; bundled avoids system SQLite version issues)
- `dirs` (for `config_dir()` / `home_dir()` resolution)

Remove:
- `tauri-plugin-opener` (not used)

Keep:
- `serde`, `serde_json`, `tauri`.

### 8.4 Cut from boilerplate

- `src-tauri/src/lib.rs`: the `greet` command and its `invoke_handler` registration.
- `src/App.tsx`: the entire boilerplate body (logos, greet form).
- `src/App.css`: the boilerplate styles.
- `src/assets/react.svg`, `public/vite.svg`, `public/tauri.svg`.

## 9. Open implementation details (decide at the keyboard)

These do not affect the spec and are safe to defer:

1. **Exact OpenRouter bar differentiator** — outline/ghost vs. neutral color vs. `$` glyph chip. Pick whichever reads cleanest at the popover's render size. (ADR-0003 explicitly leaves this open.)
2. **Codex bar color escalation thresholds** — amber at 80% / red at 95% is a sensible default; tune by eye.
3. **`rusqlite` vs. `sqlx`** — spec uses `rusqlite` (simpler, sync, fewer moving parts for a single read-only query). Switch to `sqlx` only if the async story matters (it doesn't for a 1-row read).
4. **Tray icon asset** — a real monochrome glyph is needed; the Tauri default is a placeholder.
5. **Popover exact dimensions and visual design** — the spec fixes the content (3 rows + refresh + settings), not the chrome. Apply a calm, glanceable design at implementation.

## 10. Out of scope (v1)

Explicitly deferred:

- Ambient/dynamic tray icon signal (Q9 chose static).
- Auto-launch at login (Q14 chose off).
- Backend-api Codex signal (ADR-0001 chose log-DB; backend-api is a v2 enhancement if staleness proves a problem).
- OAuth token refresh (dissolved under the log-DB decision; the Codex side doesn't touch `auth.json`).
- History/charts (a windowed view is a v2 if the popover proves useful).
- OpenRouter spend history (only current balance in v1).
- Cross-platform (Linux/Windows). macOS-only for v1; tray + `LSUIElement` are macOS-specific.
