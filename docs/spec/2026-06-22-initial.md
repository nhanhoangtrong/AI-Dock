# ai-dock — Specification

A macOS menu-bar app that surfaces AI subscription usage, rate-limit status, and credit/balance information as a glanceable status item. This document is the build spec; product reasoning and decision history live in `DESIGN.md` and `docs/adr/`.

---

## 1. Product behavior

### 1.1 App shape

A menu-bar-only application. No dock icon, no main window, no app in the Cmd-Tab switcher. Set via macOS `LSUIElement = true` (Tauri: `Activation Policy accessory` at runtime, or the `Info.plist` key).

The app exposes:
- A **tray icon** in the macOS menu bar (top-right).
- A **popover** — a hidden borderless window, shown when the tray icon is clicked, repositioned under the icon.
- A **settings affordance** inside the popover for API keys and provider visibility.

### 1.2 Tray icon

Static. No ambient signal, no fill, no ring, no badge. The icon never changes state based on data. All detail lives in the popover.

A real tray icon asset is required (the Tauri default is a placeholder). Exact design TBD; a simple monochrome glyph is sufficient for v1.

### 1.3 Popover lifecycle

- **Open:** tray icon click.
- **Close:** window blur (click elsewhere, Cmd-Tab, click another app), Escape key, or tray icon click while open (toggle).
- **Flicker guard:** a just-toggled flag prevents close-then-immediate-reopen when the tray icon is clicked while open (the click fires during the blur event).
- **No auto-timeout.** The popover stays open until the user dismisses it.

### 1.4 Popover content

Provider rows, in order:

| Row | Source | Display |
|-----|--------|---------|
| Codex Session window | `/backend-api/wham/usage` | Filled bar of `rate_limit.primary_window.used_percent` + reset timestamp |
| Codex Weekly window | `/backend-api/wham/usage` | Filled bar of `rate_limit.secondary_window.used_percent` + reset timestamp |
| Claude Code Primary window | `/api/oauth/usage` | Filled bar of `five_hour.utilization` + reset timestamp |
| Claude Code Secondary window | `/api/oauth/usage` | Filled bar of `seven_day.utilization` + reset timestamp |
| OpenRouter credits | `/api/v1/credits` | Differentiated bar (distinct treatment from Codex) + `$used / $total` text |
| DeepSeek balance | `/user/balance` | Text `$balance` only (no fill ratio — API returns remaining, not used+total) |

Plus:
- A **manual refresh button** that forces an immediate poll of all sources.
- A **settings affordance** (gear icon) that opens a panel to enter/edit the OpenRouter and DeepSeek keys and toggle provider visibility.

The provider list scrolls inside the fixed popover when every provider is visible. The footer remains pinned so refresh/settings/quit stay reachable.

### 1.5 No auto-launch

The app is not added to macOS login items. No `tauri-plugin-autostart` dependency. The user launches it manually.

## 2. Data sources

### 2.1 Codex — ChatGPT `/backend-api/wham/usage`

**Endpoint:** `GET https://chatgpt.com/backend-api/wham/usage`

**Status:** reverse-engineered and undocumented. It may change without notice. See ADR-0005.

**Auth source:** Codex CLI ChatGPT OAuth credentials from `$CODEX_HOME/auth.json`, falling back to `~/.codex/auth.json`. API-key-only Codex setups cannot show subscription usage.

**Refresh:** on `401` or `403`, refresh once via `auth.openai.com`, persist rotated tokens back into the auth file, and retry the usage request once.

**Signal:** the usage response contains current Codex subscription windows. Verified shape:

```json
{
  "plan_type": "plus",
  "rate_limit": {
    "allowed": true,
    "limit_reached": false,
    "primary_window": {
      "used_percent": 12,
      "limit_window_seconds": 18000,
      "reset_after_seconds": 17496,
      "reset_at": 1782520373
    },
    "secondary_window": {
      "used_percent": 24,
      "limit_window_seconds": 604800,
      "reset_after_seconds": 488519,
      "reset_at": 1782991396
    }
  },
  "credits": {
    "balance": "0"
  },
  "rate_limit_reset_credits": {
    "available_count": 2
  }
}
```

**Rust struct shape:**

```rust
struct CodexUsageResponse {
    plan_type: String,
    rate_limit: CodexRateLimit,
    credits: Option<CodexCredits>,
    rate_limit_reset_credits: Option<CodexResetCredits>,
}
struct CodexRateLimit {
    primary_window: CodexUsageWindow,
    secondary_window: CodexUsageWindow,
}
```

serde ignores other top-level fields (`user_id`, `account_id`, `email`, `spend_control`, `promo`, etc.) by default — no `#[serde(deny_unknown_fields)]`.

**Fresh-window normalization (required):** if a rolling window still has a full period left before reset (`reset_after_seconds` is approximately `limit_window_seconds`) and `used_percent` is 0-1%, treat it as unused. The frontend should show `0% · Not started` for the Session row until the first message starts that window.

**Fields consumed:**
- `plan_type` → badge in the popover (e.g. "Plus").
- `rate_limit.primary_window.*` → Session bar, 5h window.
- `rate_limit.secondary_window.*` → Weekly bar, 7d window.
- `rate_limit.allowed`, `rate_limit.limit_reached` → available for error/stale decisions.
- `rate_limit_reset_credits.available_count` → optional reset-credit count row.
- `credits.balance` → optional extra-usage balance row when non-zero/present.

**No local spend in option 1.** Today / Yesterday / Last 30 Days via `ccusage` are deferred.

### 2.2 Claude Code — `/api/oauth/usage`

**Endpoint:** `GET https://api.anthropic.com/api/oauth/usage`

**Status:** reverse-engineered and undocumented. It may change without notice. See ADR-0004.

**Auth:** Claude Code OAuth access token, read from Claude Code credentials. Required headers:

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer <access_token>` |
| `Accept` | `application/json` |
| `Content-Type` | `application/json` |
| `anthropic-beta` | `oauth-2025-04-20` |

**Response shape** (from OpenUsage documentation):

```json
{
  "five_hour": {
    "utilization": 25,
    "resets_at": "2026-01-28T15:00:00Z"
  },
  "seven_day": {
    "utilization": 40,
    "resets_at": "2026-02-01T00:00:00Z"
  },
  "seven_day_opus": {
    "utilization": 0,
    "resets_at": "2026-02-01T00:00:00Z"
  },
  "seven_day_omelette": {
    "utilization": 0,
    "resets_at": "2026-02-01T00:00:00Z"
  },
  "extra_usage": {
    "is_enabled": true,
    "used_credits": 500,
    "monthly_limit": 10000,
    "currency": "USD"
  }
}
```

**Fields consumed for v1:**
- `five_hour.utilization` → Claude Code 5h bar fill.
- `five_hour.resets_at` → Claude Code 5h reset timestamp.
- `seven_day.utilization` → Claude Code weekly bar fill.
- `seven_day.resets_at` → Claude Code weekly reset timestamp.
- `seven_day_opus` and `seven_day_omelette` → parse and preserve in the payload if present; not rendered in v1.
- `extra_usage` → out of v1 display scope.

**Window conversion:** convert ISO-8601 `resets_at` to unix epoch seconds. Emit the same `WindowStatus` shape as Codex rows, with `used_percent = utilization`, `window_minutes = 300` for `five_hour`, and `window_minutes = 10080` for weekly windows.

**Credential source order:**

1. macOS Keychain service `Claude Code-credentials-<sha256(CLAUDE_CONFIG_DIR).slice(0, 8)>` when `CLAUDE_CONFIG_DIR` is set.
2. macOS Keychain service `Claude Code-credentials`.
3. Fallback file `~/.claude/.credentials.json`.

**Credential shape:**

```json
{
  "claudeAiOauth": {
    "accessToken": "<jwt>",
    "refreshToken": "<token>",
    "expiresAt": 1738300000000,
    "scopes": ["..."],
    "subscriptionType": "pro",
    "rateLimitTier": "..."
  }
}
```

**Token refresh:** access tokens are short-lived. Refresh proactively 5 minutes before `expiresAt`, and reactively after `401`/`403`.

```http
POST https://platform.claude.com/v1/oauth/token
Content-Type: application/json
```

```json
{
  "grant_type": "refresh_token",
  "refresh_token": "<refresh_token>",
  "client_id": "9d1c250a-e61b-44d9-88ed-5944d1962f5e",
  "scope": "user:profile user:inference user:sessions:claude_code user:mcp_servers user:file_upload"
}
```

Persist refreshed credentials back to the same source when possible.

**No settings key field.** This provider uses Claude Code OAuth credentials, not an Anthropic API key. Settings should expose only the provider visibility toggle, with hint text in the provider label, e.g. `Claude Code (OAuth from Keychain)`.

### 2.3 OpenRouter — `/api/v1/credits`

**Endpoint:** `GET https://openrouter.ai/api/v1/credits`

**Auth:** `Authorization: Bearer <management_key>`. The docs flag this as a management-key endpoint; a standard chat `sk-or-...` key may be rejected with 401/403. The error policy (§4.3) handles this.

**Response shape** (verified from OpenRouter docs):

```json
{
  "data": {
    "total_credits": 100.5,
    "total_usage": 25.75
  }
}
```

**Derived value:** remaining = `total_credits - total_usage` (the API does not return a "remaining" field).

**HTTP client:** `reqwest` async.

**Timeout:** 10 seconds. Failures fall through to the error policy (§4.3).

### 2.4 DeepSeek — `/user/balance`

**Endpoint:** `GET https://api.deepseek.com/user/balance`

**Auth:** `Authorization: Bearer <api_key>` (a regular DeepSeek API key from platform.deepseek.com).

**Response shape:**

```json
{
  "is_available": true,
  "balance_infos": [
    {
      "currency": "USD",
      "total_balance": "4.12",
      "granted_balance": "0.00",
      "topped_up_balance": "4.12"
    }
  ]
}
```

**Parsing:** `total_balance` is a string; parse as `f64`. Sum across all currencies (typically single USD). If `is_available` is `false`, return an `Error` with "account unavailable".

**Display:** no meaningful fill ratio (the API returns only remaining balance, not total+used). The row shows just `$total_balance` as text, with no bar.

**HTTP client:** same `reqwest` client as OpenRouter.

**Timeout:** 10 seconds.

**Fields consumed:**
- `total_balance` → the dollar amount to display.
- `currency` → `"USD"` → prefix with `$`; otherwise show the currency code before the number.
- `is_available` → if false, surface as an error.

## 3. Credential storage

**Path:** `~/.config/ai-dock/config.json` (resolved via `dirs::config_dir()`).

**Format:** JSON (serde_json is already a Tauri dependency; no new crate for TOML).

**Shape:**

```json
{
  "openrouter_key": "sk-or-...",
  "deepseek_key": "sk-...",
  "provider_visibility": {
    "codex": true,
    "claude": true,
    "openrouter": true,
    "deepseek": true
  }
}
```

**Behavior:**
- Created lazily on first write (when the user enters a key in settings). Not created at launch if absent.
- Read by Rust at poll time. The key never crosses to the frontend webview.
- Hand-editable; the app re-reads on each poll (no in-memory cache that goes stale after an external edit).
- Missing file or missing field → treated as a persistent error ("no OpenRouter key — add one in settings").
- `provider_visibility` is generic and keyed by provider id. Missing provider visibility keys default to `true`, so new providers appear by default until explicitly hidden.
- Claude Code credentials are **not** stored in this config file. They are read from Claude Code OAuth credential storage (§2.2).

## 4. Error and stale states

**Invariant:** never render a number we don't have. Zero is a number. A failed fetch is not zero and must not be rendered as zero.

### 4.1 Codex failure modes

| Failure | Display |
|---------|---------|
| Codex auth file missing | Error caption: "Codex: not logged in — run `codex` once." No bar. |
| Codex auth mode is API-key-only | Error caption: "Codex: ChatGPT login required." No bar. |
| Access token rejected after one refresh retry | Error caption: "Codex: sign in again." No bar. |
| Usage API rate-limited | Error caption: "Codex: rate-limited, retrying." No bar. |
| Usage response parse fail / schema changed | Error caption: "Codex: usage format changed — check for app update." No bar. |

### 4.2 Claude Code failure modes

| Failure | Display |
|---------|---------|
| No Keychain or fallback credentials | Error caption: "Claude Code: no credentials found — run Claude Code once." No bar. |
| Keychain/file read failure | Error caption: "Claude Code: could not read credentials." No bar. |
| Token refresh failure | Error caption: "Claude Code: sign in again." No bar. |
| 401/403 after refresh | Error caption: "Claude Code: sign in again." No bar. |
| 429 (rate-limited by API) | Dim last-known bar, caption: "Claude Code: rate-limited, retrying." Retry next poll. |
| Network error / timeout / 5xx | Dim last-known bar, caption: "Claude Code: couldn't fetch." Retry next poll. |
| Malformed JSON / unexpected response shape | Error caption: "Claude Code: usage format changed — check for app update." No bar. |

### 4.3 OpenRouter failure modes

| Failure | Display |
|---------|---------|
| 401/403 (key missing, wrong, or chat key not management key) | Error caption: "OpenRouter: key rejected — is it a management key?" No bar. |
| 429 (rate-limited by OpenRouter) | Dim last-known bar, caption: "OpenRouter: rate-limited, retrying." Retry next poll. |
| Network error / timeout / 5xx | Dim last-known bar, caption: "OpenRouter: couldn't fetch." Retry next poll. |
| Malformed JSON | Dim last-known bar, caption: "OpenRouter: unexpected response." Retry next poll. |
| No key configured | Error caption: "OpenRouter: no key — add one in settings." No bar. |

### 4.4 DeepSeek failure modes

| Failure | Display |
|---------|---------|
| 401/403 (key missing or wrong) | Error caption: "DeepSeek: key rejected — check your API key." No bar. |
| 429 (rate-limited by DeepSeek) | Error caption: "DeepSeek: rate-limited, retrying." No bar. |
| Network error / timeout / 5xx | Error caption: "DeepSeek: couldn't fetch." No bar. |
| Malformed JSON / unexpected response | Error caption: "DeepSeek: unexpected response." No bar. |
| `is_available` is `false` | Error caption: "DeepSeek: account unavailable." No bar. |
| No key configured | Error caption: "DeepSeek: no key — add one in settings." No bar. |

### 4.5 First-launch state (no prior good values)

When there has never been a good value for a source, there is nothing to show stale. The row renders the error caption only, no bar. This is the honest first-run experience: the popover shows error captions for each unconfigured source and a settings prompt.

## 5. Architecture

### 5.1 Rust owns the poll loop

A background `tokio` task polls every **5 minutes** (300 seconds), same interval for all sources. Each cycle:

1. Read `~/.config/ai-dock/config.json` → if keys present, fetch OpenRouter credits and DeepSeek balance.
2. Read Codex ChatGPT OAuth credentials → fetch Codex usage.
3. Read Claude Code OAuth credentials → fetch Claude Code usage.
4. Run provider fetches concurrently via `tokio::join!`.
5. Build a combined `Status` payload (codex + claude + openrouter + deepseek).
6. Emit a `status-update` Tauri event to the frontend.

The frontend never polls, never fetches, never reads files. It subscribes to `status-update` and renders.

### 5.2 Manual refresh

A Tauri command `refresh_now()` that the frontend invokes from the refresh button. It triggers an immediate poll cycle (same logic as the timer) and emits `status-update`. The 5-min timer is not reset by a manual refresh (the next auto-poll fires on its own schedule).

### 5.3 Tray + popover plumbing

- `TrayIconBuilder` creates the tray icon with the static asset.
- On tray click: if the popover window is hidden, reposition it under the tray icon and show; if visible, hide. (Toggle.)
- The popover window is created at app launch with `visible: false`, borderless, no decorations, `always_on_top`, `skip_taskbar` (Tauri equivalents).
- Repositioning under the tray icon: query the tray icon's position (Tauri API) and set the window position so the popover appears below it, account for the popover's width/height.
- Hide-on-blur: `window.on_focus_changed` → hide when focus lost, *unless* the just-toggled flag is set (set on tray click, cleared after a short delay or the next event loop tick).

### 5.4 macOS-specific config

- `LSUIElement = true` (or `Activation Policy accessory` at runtime) — no dock icon, no Cmd-Tab entry.
- The window is borderless (`decorations: false`) and transparent-background-capable if the popover design needs rounded corners / a shadow.

## 6. Payload contract (Rust → frontend)

The `status-update` event payload:

```ts
type StatusUpdate = {
  codex: CodexStatus;
  claude: ClaudeStatus;
  openrouter: OpenRouterStatus;
  deepseek: DeepSeekStatus;
  polled_at: number; // unix epoch seconds
};

type CodexStatus =
  | { kind: "ok"; plan_type: string; primary: WindowStatus; secondary: WindowStatus; event_ts: number }
  | { kind: "error"; message: string }
  | { kind: "stale"; plan_type: string; primary: WindowStatus; secondary: WindowStatus; event_ts: number; message: string };

type WindowStatus = {
  used_percent: number;  // zeroed by Rust if reset_at <= now (window already rolled)
  window_minutes: number;
  reset_after_seconds: number;  // zeroed by Rust if reset_at <= now
  reset_at: number; // unix epoch seconds
};

type ClaudeStatus =
  | {
      kind: "ok";
      five_hour: WindowStatus;
      seven_day: WindowStatus;
      seven_day_opus?: WindowStatus;
      seven_day_omelette?: WindowStatus;
    }
  | { kind: "error"; message: string }
  | {
      kind: "stale";
      five_hour: WindowStatus;
      seven_day: WindowStatus;
      seven_day_opus?: WindowStatus;
      seven_day_omelette?: WindowStatus;
      message: string;
    };

type OpenRouterStatus =
  | { kind: "ok"; total_credits: number; total_usage: number; remaining: number }
  | { kind: "error"; message: string }
  | { kind: "stale"; total_credits: number; total_usage: number; remaining: number; message: string };

type DeepSeekStatus =
  | { kind: "ok"; total_balance: number; currency: string }
  | { kind: "error"; message: string }
  | { kind: "stale"; total_balance: number; currency: string; message: string };
```

`kind` discriminates the three display states (§4): `ok` (normal), `stale` (dim + caption), `error` (caption only, no bar).

## 7. Frontend

### 7.1 Stack

React 19 + TypeScript + Vite, kept from the boilerplate. No new frontend dependencies for v1.

### 7.2 Components

```
src/
  App.tsx                      # subscribes to status-update, renders popover shell
  App.css                      # popover styles (replaces boilerplate)
  components/
    Bar.tsx                    # the progress bar; prop: variant ("filled" | "differentiated"), fill (0-100)
    Row.tsx                    # one row: label + bar (+ badge) + secondary text + caption (stale/error).
                               # Supports `fill: undefined` for rows that don't render a bar (DeepSeek).
    Settings.tsx               # key entry panel for OpenRouter and DeepSeek keys; provider visibility toggles.
  providers.ts                 # provider ids/labels and visibility defaults
```

### 7.3 Bar variants

- **Codex rows:** `variant="filled"`. A solid filled bar, color escalates near the cap (thresholds TBD at implementation; default: amber at 80%, red at 95%).
- **Claude Code rows:** `variant="filled"`. Real subscription utilization bars for 5h and weekly windows. Badge: `subscription`.
- **OpenRouter row:** `variant="differentiated"`. Same shape (a bar), distinct treatment — outline/ghost or neutral color, TBD at implementation per ADR-0003. Fill ratio = `total_usage / total_credits`.
- **DeepSeek row:** `variant="differentiated"` but no bar rendered (fill is `undefined`). The DeepSeek API returns only remaining balance, not used+total, so there's no meaningful fill ratio. Shows just `$balance` as text.

### 7.4 Provider visibility

Provider visibility is controlled by a generic `provider_visibility` config object. Known providers in v1:

| Provider id | Label |
|-------------|-------|
| `codex` | `Codex` |
| `claude` | `Claude Code (OAuth from Keychain)` |
| `openrouter` | `OpenRouter` |
| `deepseek` | `DeepSeek` |

Missing visibility keys default to visible. Settings renders toggles from the shared provider list so future providers require one provider-list entry plus rows/payload handling, not a bespoke settings section. Hints belong in labels, not separate hint blocks.

### 7.5 Event subscription

`App.tsx` listens to the Tauri `status-update` event on mount, stores the latest payload in state, and passes slices to the rows. On unmount, unsubscribe.

### 7.6 Refresh button

Calls `invoke("refresh_now")`. The actual UI update comes through the same `status-update` event (the manual refresh triggers a poll that emits the event); the button does not directly update state.

## 8. Tauri configuration changes

### 8.1 `tauri.conf.json`

- `"productName"`: keep `ai-docked` or rename to `ai-dock` (consistency with repo name).
- `"app.windows[0]"`: set `"visible": false`, `"decorations": false`, `"alwaysOnTop": true`, `"skipTaskbar": true`, `"resizable": false`. Reduce `width`/`height` to popover dimensions (e.g. 320 × 240).
- Add tray icon configuration if Tauri 2 supports it via config (otherwise build in Rust via `TrayIconBuilder`).

### 8.2 `capabilities/default.json`

- Add permission for the `tray` plugin / tray icon if required by the Tauri 2 tray API.
- Add permission to emit events to the window (`core:event:allow-emit` or equivalent).
- Remove `opener:default` (we're cutting `tauri-plugin-opener`).

### 8.3 `Cargo.toml` — dependency changes

Add:
- `tokio` (with `rt-multi-thread`, `time` features — for the poll interval)
- `reqwest` (default features or `json` feature — for Codex, OpenRouter, DeepSeek, and Claude Code fetches)
- `dirs` (for `config_dir()` / `home_dir()` resolution)
- `sha2` (for Claude Code config-specific Keychain service names)

Remove:
- `tauri-plugin-opener` (not used)

Keep:
- `serde`, `serde_json`, `tauri`.

### 8.4 Cut from boilerplate

- `src-tauri/src/lib.rs`: the `greet` command and its `invoke_handler` registration.
- `src/App.tsx`: the entire boilerplate body (logos, greet form).
- `src/App.css`: the boilerplate styles.
- `src/assets/react.svg`, `public/vite.svg`, `public/tauri.svg`.

## 9. Open implementation details (decide at the keyboard)

These do not affect the spec and are safe to defer:

1. **Exact OpenRouter bar differentiator** — outline/ghost vs. neutral color vs. `$` glyph chip. Pick whichever reads cleanest at the popover's render size. (ADR-0003 explicitly leaves this open.)
2. **Codex bar color escalation thresholds** — amber at 80% / red at 95% is a sensible default; tune by eye.
3. **Codex reset-credit expiry details** — option 1 surfaces the embedded count only. The dedicated expiry endpoint is deferred.
4. **Tray icon asset** — a real monochrome glyph is needed; the Tauri default is a placeholder.
5. **Popover exact dimensions and visual design** — the spec fixes the content (provider rows + refresh + settings), not the chrome. Apply a calm, glanceable design at implementation.
6. **DeepSeek bar treatment** — the row shows text only (no bar). The `variant="differentiated"` label is reserved for consistency but unused at render.

## 10. Out of scope (v1)

Explicitly deferred:

- Ambient/dynamic tray icon signal (Q9 chose static).
- Auto-launch at login (Q14 chose off).
- Codex spend tiles through `ccusage` (Today / Yesterday / Last 30 Days).
- Dedicated Codex reset-credit expiry tooltip through `/backend-api/wham/rate-limit-reset-credits`.
- History/charts (a windowed view is a v2 if the popover proves useful).
- OpenRouter spend history (only current balance in v1).
- Cross-platform (Linux/Windows). macOS-only for v1; tray + `LSUIElement` are macOS-specific.
